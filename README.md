# Apple Event HLS 视频下载器

可靠的 Apple Event 视频下载工具，使用经过验证的分段下载方法。

## ✨ 特性

- ✅ **健壮可靠** - 每个分段最多重试5次
- ✅ **断点续传** - 中断后重新运行会跳过已下载的分段
- ✅ **文件验证** - 自动验证文件大小，拒绝损坏的分段
- ✅ **实时进度** - 显示下载进度和统计信息
- ✅ **完整性检查** - 下载后自动验证视频时长

## 🚀 快速开始

### 1. 测试下载（10分钟）

先测试方法是否有效：

```bash
./download_apple_video.sh --test
```

完成后会生成 `apple_event_test_10min.mp4`，用播放器验证是否完整。

### 2. 完整下载（72分钟）

测试成功后下载完整视频：

```bash
./download_apple_video.sh
```

**预计时间**: 30-60 分钟（取决于网络速度）

**输出文件**: `apple_event_complete.mp4`

### 3. 验证视频

```bash
./verify_video.sh apple_event_complete.mp4
```

看到 `✅ 视频完整！时长匹配` 就说明成功了！

## ⚙️ 高级用法

### 指定下载分段数量

```bash
# 下载前30分钟（300个分段）
./download_apple_video.sh --segments 300
```

### 断点续传

如果下载中断，直接重新运行即可：

```bash
./download_apple_video.sh
```

脚本会自动：
- 跳过已下载的有效分段
- 只下载缺失或损坏的分段
- 继续之前的进度

## 📊 下载统计

脚本会显示实时统计：

```
[50%] 359/719 - 已下载: 1.8G (成功:300 跳过:59 失败:0)
```

- **成功**: 本次新下载的分段
- **跳过**: 已存在的有效分段
- **失败**: 下载失败的分段（会重试5次）

## ⚠️ 常见问题

### Q: 下载速度慢？

A: 脚本使用顺序下载（一个接一个），确保可靠性。每个分段约6秒视频。

### Q: 部分分段失败？

A: 重新运行脚本即可，会自动重试失败的分段。

### Q: 如何更换画质？

A: 编辑 `download_apple_video.sh`，修改这行：

```bash
VIDEO_STREAM="sdr_avc_1080p_8500"  # 改为其他画质
```

可选项：
- `sdr_avc_720p_4500` - 720p（更小）
- `sdr_avc_1080p_8500` - 1080p（推荐）
- `sdr_hvc_2160p_16800` - 4K（更大）

### Q: 视频有黑屏？

A: 运行验证工具检查：

```bash
./verify_video.sh apple_event_complete.mp4
```

如果时长不匹配，说明部分分段下载失败，重新运行下载脚本。

## 🛠️ 技术细节

### 为什么不用 ffmpeg 直接下载？

ffmpeg 在网络不稳定时容易中断，导致只下载部分视频分段，但生成完整时长的文件（音频完整，视频部分黑屏）。

### 本工具的方法

1. 逐个下载每个视频分段（.m4s文件）
2. 每个分段验证文件大小（必须>100KB）
3. 失败自动重试（最多5次）
4. 手动拼接所有分段
5. 使用 ffmpeg 混流视频+音频

这个方法虽然慢，但**100%可靠**。
