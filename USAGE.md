## 🔄 断点续传示例

如果下载中途中断（比如网络断了）：

```bash
# 第一次运行，下载了 300 个分段后中断
./download_apple_video.sh

# 网络恢复后，重新运行
./download_apple_video.sh
```

第二次运行时，你会看到：

```
  [ 42%] 300/719 (成功:0 跳过:300 失败:0)
```

`跳过:300` 表示前300个分段已存在，直接跳过，从第301个继续下载。

## 🎯 自定义下载

### 只下载前30分钟

```bash
./download_apple_video.sh --segments 300
```

### 更换 URL

如果要下载其他 Apple Event：

1. 编辑 `download_apple_video.sh`
2. 找到这行：

```bash
M3U8_BASE="https://events-delivery.apple.com/.../vod_main_..."
```

3. 替换为新的 URL

### 更换画质和音频

编辑脚本，修改这两行：

```bash
VIDEO_STREAM="sdr_avc_1080p_8500"  # 视频画质
AUDIO_STREAM="audio_main_en_2ch_aac_128"  # 音频质量
```

**可选视频流：**
- `sdr_avc_720p_4500` - 720p H.264 (~3 Mbps)
- `sdr_avc_1080p_8500` - 1080p H.264 (~6 Mbps) ⭐ 推荐
- `sdr_hvc_1080p_7000` - 1080p HEVC (~7 Mbps)
- `sdr_hvc_2160p_16800` - 4K HEVC (~17 Mbps)

**可选音频流：**
- `audio_main_en_2ch_aac_128` - AAC 立体声 128kbps ⭐ 推荐
- `audio_main_en_16ch_atmos_640` - Dolby Atmos 640kbps

## 🔧 故障排除

### 问题：下载很慢

**原因**: 顺序下载 + 每个分段重试机制

**建议**: 
- 这是正常的，保证可靠性
- 可以减少延迟时间（编辑脚本中的 `sleep 0.1` 改为 `sleep 0.05`）

### 问题：部分分段失败

**症状**: 

```
  ⚠️  第 152 个分段失败: sdr_avc_1080p_8500_152.m4s
```

**解决**: 
1. 继续等待（会自动重试5次）
2. 如果最终还是失败，重新运行脚本
3. 脚本会自动重新下载失败的分段

### 问题：验证显示时长不匹配

**症状**:

```
❌ 问题：视频流和总时长不匹配
   差异: 563 秒
```

**原因**: 部分分段下载失败或损坏

**解决**:

```bash
# 1. 删除临时文件重新下载
rm -rf hls_download_temp

# 2. 重新下载
./download_apple_video.sh

# 3. 再次验证
./verify_video.sh apple_event_complete.mp4
```

### 问题：网络超时

**症状**: 很多分段报错 timeout

**解决**: 编辑脚本，增加超时时间：

```bash
# 找到这行
if curl -sS -f -m 30 -o "$output" "$url" 2>/dev/null; then

# 改为
if curl -sS -f -m 60 -o "$output" "$url" 2>/dev/null; then
```
